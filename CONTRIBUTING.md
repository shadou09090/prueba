# Contributing to Simple Interest Calculator

Thank you for your interest in contributing to this project! To maintain a clean and organized codebase, please follow these guidelines when making contributions.

## How to Contribute

### 1. Reporting Bugs
If you find a bug or an error in the documentation, please open an **Issue** in the repository. Include:
* A clear and descriptive title.
* Steps to reproduce the issue.
* Expected behavior vs. actual behavior.
* Screenshots if applicable.

### 2. Suggesting Enhancements
If you have ideas to improve the calculator or the documentation:
* Open an **Issue** describing the enhancement.
* Explain why this change would be useful to the project.

### 3. Submitting Changes (Git Workflow)
To contribute code or documentation fixes directly, please follow these steps:

1. **Fork** this repository to your own GitHub account.
2. **Clone** the forked repository to your local machine.
3. Create a new **branch** for your changes. Use a descriptive name:
   `git checkout -b bug-fix-typo` or `git checkout -b feature-new-formula`
4. Make your changes in the appropriate files.
5. **Commit** your changes with a clear and meaningful commit message:
   `git commit -m "Fix typo in README footer"`
6. **Push** the changes to your remote branch on GitHub:
   `git push origin your-branch-name`
7. Open a **Pull Request (PR)** from your branch to the `main` branch of the original repository.

## Code Style and Standards
* Keep the code clean, well-commented, and easy to understand.
* Ensure markdown files use proper formatting and headers.
* Test your changes locally before submitting a Pull Request.
